# Оптимізація розміру проєкту

## Проблеми з розміром

### Виявлені проблеми:

1. **`.cache` директорія** (143 MB) ✅ ВИДАЛЕНО
   - Тимчасовий кеш від webpack/react-scripts
   - Автоматично регенерується при запуску
   - Додано в `.gitignore`

2. **Відео файл `bg-video.mp4`** (63.29 MB) ⚠️ ПОТРЕБУЄ ОПТИМІЗАЦІЇ
   - Дуже великий розмір для веб-використання
   - Рекомендації:
     - Стиснути відео до меншого розміру (наприклад, до 5-10 MB)
     - Використати формат WebM замість MP4 для кращого стиснення
     - Зменшити роздільність якщо можливо
     - Використати інструменти типу HandBrake або FFmpeg

3. **TypeScript** (63.75 MB) ℹ️ НЕ МОЖНА ВИДАЛИТИ
   - Встановлений як залежність від `react-scripts`
   - Не використовується безпосередньо в проєкті
   - Це нормально для Create React App проєктів

## Рекомендації для оптимізації:

### 1. Оптимізація відео
```bash
# Використати FFmpeg для стиснення
ffmpeg -i bg-video.mp4 -vcodec libvpx-vp9 -crf 30 -b:v 0 -b:a 128k -c:a libopus bg-video.webm

# Або зменшити роздільність та бітрейт
ffmpeg -i bg-video.mp4 -vf scale=1280:720 -b:v 1M -b:a 128k bg-video-optimized.mp4
```

### 2. Регулярне очищення кешу
```bash
# Видалити .cache перед деплоєм
rm -rf client/node_modules/.cache
```

### 3. Використання CDN для великих файлів
- Розглянути використання CDN для відео файлів
- Або використати YouTube/Vimeo для хостингу відео

### 4. Production build оптимізація
- Використовувати `npm run build` для production
- Build файли вже оптимізовані та стиснуті
- Розмір build директорії: ~5.42 MB (нормально)

## Поточний стан:

- ✅ `.cache` видалено та додано в `.gitignore`
- ⚠️ Відео потребує оптимізації (63 MB → можна зменшити до 5-10 MB)
- ℹ️ TypeScript залишається (залежність від react-scripts)

## Очікуваний результат після оптимізації відео:

- Загальний розмір: ~380 MB (замість ~524 MB)
- Зменшення на ~144 MB (27%)
