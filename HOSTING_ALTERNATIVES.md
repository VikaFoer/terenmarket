# Альтернативи для запуску проєкту

## Проблема
На поточному тарифі хостингу Node.js недоступний. Потрібен "Бізнес-хостинг".

## Рішення

### Варіант 1: Змінити тариф на Бізнес-хостинг (найпростіше)

1. Зайдіть в панель ADM.TOOLS
2. Знайдіть розділ "Тарифи" або "Плани"
3. Оновіть тариф до "Бізнес-хостинг"
4. Після активації Node.js стане доступним
5. Встановіть Node.js через панель
6. Запустіть проєкт

**Переваги:**
- ✅ Все на одному хостингу
- ✅ Просто налаштувати
- ✅ Підтримка хостингу допоможе

**Недоліки:**
- ❌ Додаткові витрати

### Варіант 2: Railway (БЕЗКОШТОВНО для початку) ⭐ РЕКОМЕНДОВАНО

Railway вже налаштований в проєкті! Це найпростіший варіант.

**Як задеплоїти:**

1. Зареєструйтеся на [railway.app](https://railway.app) (через GitHub)

2. Створіть новий проєкт:
   - Натисніть "New Project"
   - Оберіть "GitHub Repo"
   - Виберіть `VikaFoer/terenmarket`

3. Railway автоматично:
   - Встановить залежності
   - Збере фронтенд
   - Запустить сервер

4. Налаштуйте змінні оточення:
   - Settings → Variables
   - Додайте `.env` змінні

5. Створіть Volume для бази даних:
   - Volumes → New Volume
   - Mount Path: `/app/server/data`
   - Size: 5 GB

**Переваги:**
- ✅ Безкоштовно для початку ($5 кредит щомісяця)
- ✅ Автоматичний деплой з GitHub
- ✅ HTTPS автоматично
- ✅ Проєкт вже налаштований для Railway

**Інструкції:** Дивіться `RAILWAY_DEPLOYMENT_GUIDE.md` або `README.md`

### Варіант 3: VPS сервер (якщо є)

Якщо у вас є VPS сервер:

1. Підключіться до VPS
2. Встановіть Node.js:
   ```bash
   curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
   sudo apt-get install -y nodejs
   ```
3. Завантажте проєкт через git
4. Запустіть `./setup-server.sh`

### Варіант 4: Heroku (альтернатива Railway)

1. Зареєструйтеся на [heroku.com](https://heroku.com)
2. Встановіть Heroku CLI
3. Задеплойте:
   ```bash
   heroku create tecsamarket
   git push heroku main
   ```

### Варіант 5: Vercel (тільки фронтенд) + окремий backend

Якщо потрібно розділити:
- Frontend на Vercel (безкоштовно)
- Backend на Railway або іншому сервісі
